Event emitters (see on github)

Event binding notes:

  • Angular Event emitters is one way in which child and parent components can communicate.
  • Data flows into a component via property bindings and flows out of the component through event bindings.
  • A component can comminicate with its parent by using the Output decorator with EventEmitter to create a custom event.
  • Unlike DOM events Angular custom EventEmitters do not bubble.
  • EventEmitter are synchronous by default but then can accept a boolean parameter to be asynchronous.
  • Because EventEmitters are Subjects ,we can use all the Rx features.
  • In parent child component communication property binding is used to send data from parent to child component.
  • In parent child component communication custom event binding is used to send data from the child to the parent component.

Usage guidelines:

  • To create a custom event create an instance of EventEmitter annotated by @Output().
    Ex: @Output() statusChanged = new EventEmitter<{ id: number, newStatus: string }>();
  • The EventEmitter instance calls the emit() method to send data which can be received by event object $event.
    Ex (in HTML): (statusChanged)="onStatusChanged($event)"
  • The @Output decorator tell angular that the property (the event emitter in this case) is available for parent components.
  • The @Output decorator enable outside components to listen to a custom event, which is created using the EventEmitter.
  • The @Input and @output decorator are more appropriate for components that have direct child-parent relationship.
  • In cases where the distance between the component is greater, it's better to avoid chain of events emitting/receuving and use other methods, such as services.
